# AI Drafting UI Plan

## Goal

Add explicit user control for AI-generated reply drafts:

- Default inbound behavior: do not call the LLM for new emails.
- Settings UI: add an `AI Drafting` section with `Automatically create reply drafts for new emails` toggle.
- Composer UI: add `Generate Draft` for reply mode only.
- Manual generation: show a preview first, then apply to the composer only when the user confirms.

## Recommended Data Model

Use a new mailbox settings namespace instead of the existing unused `autoReply` field.

```ts
aiDrafting?: {
	autoCreateReplyDrafts?: boolean;
};
```

Interpret missing values as `false`. Also add `aiDrafting: { autoCreateReplyDrafts: false }` to new mailbox defaults for clarity.

## Backend Plan

1. Add `aiDrafting.autoCreateReplyDrafts` to mailbox defaults in `workers/index.ts`.

2. Gate inbound auto-drafting in `receiveEmail()` before `EMAIL_AGENT /onNewEmail` is called.

   - Read `mailboxes/${mailboxId}.json` from R2.
   - Continue only if `settings.aiDrafting?.autoCreateReplyDrafts === true`.
   - If missing or false, skip the agent call entirely.
   - This is the key cost/privacy control point because it prevents the LLM path from starting.

3. Add a manual preview endpoint near the existing reply/forward routes.

   Recommended route:

   ```txt
   POST /api/v1/mailboxes/:mailboxId/emails/:id/reply-draft-preview
   ```

   Response shape:

   ```ts
   {
	to: string;
	subject: string;
	body: string;
   }
   ```

4. Implement preview generation through `EmailAgent` without saving a Draft.

   - Add an internal agent route such as `/generateReplyDraftPreview` in `EmailAgent.onRequest()`.
   - Add `handleGenerateReplyDraftPreview({ mailboxId, emailId })`.
   - Reuse the same model and safety approach as `handleNewEmail()`:
	 - load original email and thread context;
	 - run `isPromptInjection()` on email/thread context;
	 - call `generateText()` with a prompt that asks for reply body only;
	 - run `verifyDraft()` on the returned body;
	 - return preview data only.
   - Do not call `toolDraftReply()` because that saves to Drafts as a side effect.

5. Keep automatic and manual generation separate.

   - Automatic path continues to save to Drafts only when enabled.
   - Manual path returns a preview only and never sends or saves automatically.

## Frontend Plan

1. Settings UI in `app/routes/settings.tsx`.

   - Add local state: `autoCreateReplyDrafts`.
   - Load from `mailbox.settings?.aiDrafting?.autoCreateReplyDrafts`.
   - Save under:

   ```ts
   aiDrafting: {
	...mailbox.settings?.aiDrafting,
	autoCreateReplyDrafts,
   }
   ```

   - Add a card titled `AI Drafting`.
   - Add copy: `Automatically create reply drafts for new emails`.
   - Include helper copy clarifying that drafts are created for review and never sent automatically.

2. API client and mutation.

   - Add `api.generateReplyDraftPreview(mailboxId, emailId)` in `app/services/api.ts`.
   - Add `useGenerateReplyDraftPreview()` in `app/queries/emails.ts`.

3. Shared composer hook in `app/hooks/useComposeForm.ts`.

   Add state and actions:

   ```ts
   draftPreview: string | null;
   isGeneratingDraft: boolean;
   generateDraftError: string | null;
   handleGenerateDraftPreview(): Promise<void>;
   handleApplyDraftPreview(): void;
   handleDismissDraftPreview(): void;
   ```

   Show generation only when:

   - `composeOptions.mode === "reply"` or existing reply mode equivalent;
   - `composeOptions.originalEmail?.id` exists;
   - not editing an existing draft.

4. Composer UI.

   - In `ComposePanel.tsx`, place `Generate Draft` in the top header action area next to the close button.
   - In `ComposeEmail.tsx`, place the same action near the modal title or the first row of the form.
   - Show a loading state while generating.
   - Show a preview panel after generation with actions:
	 - `Apply`
	 - `Dismiss`
	 - optionally `Regenerate` if the same handler is simple to reuse.

5. Apply behavior.

   - Do not overwrite the composer until `Apply` is clicked.
   - On apply, insert the generated reply body into the editor.
   - Preserve existing quoted reply block/signature where possible. If the current reply body already contains a quote block, replace only the editable reply portion.

## UX Copy

Settings:

- Section title: `AI Drafting`
- Toggle label: `Automatically create reply drafts for new emails`
- Helper: `When enabled, the agent reads new inbound emails and creates reply drafts for review. Drafts are never sent automatically.`

Composer:

- Button: `Generate Draft`
- Preview title: `AI draft preview`
- Apply button: `Apply to message`
- Dismiss button: `Dismiss`

## Safety And Cost Notes

- Default is off, so new inbound email does not call the LLM unless the user enables it.
- Manual generation only calls the LLM after an explicit user click.
- Prompt injection checks should run for both automatic and manual generation.
- Preview generation must not send email and must not save to Drafts.
- Disable the button while generation is in flight to avoid duplicate calls.

## Verification

1. Run type checks:

   ```bash
   npm run typecheck
   ```

2. Settings behavior:

   - Open mailbox settings.
   - Confirm `AI Drafting` toggle defaults off for existing and new mailboxes.
   - Toggle on, save, reload, confirm persistence.
   - Toggle off, save, reload, confirm persistence.

3. Inbound behavior:

   - With setting off, simulate/receive a new email and confirm `EMAIL_AGENT /onNewEmail` is not called and no AI draft is created.
   - With setting on, confirm the existing automatic draft flow still creates a Drafts item.

4. Composer behavior:

   - Reply to an email and confirm `Generate Draft` appears.
   - New email and forward flows should not show the button.
   - Click `Generate Draft`; confirm preview appears and body is not changed yet.
   - Click `Apply to message`; confirm the editor body updates.
   - Confirm send/save draft still work after applying.

5. Safety behavior:

   - Use a prompt-injection-like email and confirm manual generation is rejected without changing the composer.
   - Confirm automatic generation is skipped for the same kind of content.
